Background

In this hands-on lab, you'll create process to automate the updating of an invoice in Oracle Engagement Cloud.

We'll base this lab on the subscription billing demo discussed during other Oracle OpenWorld sessions, and on the previous lab where you created an integration.

The objective is to automate the update process to bill subscriptions for use of assets. The process you'll create will let you update the number of subscriptions for a particular product.

Sign In

Access to Oracle Integration is secured through Oracle Identity Cloud Service. Users may be granted access to all the Oracle Integration environments in a cloud tenancy, or they may be restricted to specific environments. They may be assigned different roles in each environment.

  1. Click the Oracle Integration bookmark on your browser to open the Oracle Integration instance.

  2. Enter your user name and password.

    • Your user name is OICTraining1ic
      This is the prefix OICTraining1ic followed by the user number you were assigned.
    • Your password is listed in the document on your desktop.
    • Oracle Integration will log all your design activities with this user name.

    The Welcome page appears. Check out the tour, videos, and links to learn more. Keep scrolling to find more useful links.

  3. Click Open Home.

    The Home page displays an overview with the current state of your instance. You can drill down to get information about available components, tasks, and instances.

Create a Process

You’ll start with a predefined process and make changes to it.

  1. From the Home page’s Navigation menu, select Processes, and then select Process Applications.
  2. Click Create.
  3. Hover over the Start with a QuickStart box, and then click Browse.

  4. Locate the Subscription Management QuickStart, and click its Create button.

  5. In the Create Application dialog, in the name field, enter My Subscriptions.

  6. Click Create.

    The quickstart application appears.

Edit the Form

You’ll add two controls to the form that each call the source system and return values for customers to choose from.

  1. On the QuickStart App page, click the Switch to Application View link.

  2. On the Processes page, click the process to open it in the structured process editor.
  3. Click the purple Start activity titled Submit Subscription Update.

  4. Click and select Open Properties.

  5. In the How will the end user identify the process? section, in the Title field, enter User Submit Subscription Update.

    Remember this name for later, when you run the process application.

  6. Close the Properties pane and select the same activity again.

  7. Click and select Open Form.

    The form editor appears. It contains a central canvas, a side pane for setting properties for selected controls, and a side palette pane with basic and advanced controls you can add to the form.

  8. Add a Select control to select the subscription:

    1. Drag and drop a Select control from the Basic Palette to the top of the form.

    2. Select the control you just added.

      Notice how the Properties pane changes: when you select a control, the properties panel displays its properties. When you click outside of a control, the properties panel displays the entire form’s properties.

    3. In the Properties pane, locate the General tab.
    4. In the Name field, enter SubscriptionPUID.
    5. In the Label field, enter Subscription.

    6. Click outside the new control.

      Notice how the label is now Subscription and the select control is a dropdown field.

  9. Add a Select control to select the product:

    1. In the form editor, drag and drop a Select control below the one you just created.
    2. Select the control you just added.
    3. In the Properties pane, locate the General tab.
    4. in the Name field, enter ProductPUID.

    5. In the Label field, enter Product.

  10. Click the Preview button to see the new fields in your form.
  11. Click to close the preview.

Link the Integrations

Here you’ll add two active integrations for use in your process.

  1. From the Navigation menu, select Integrations.
  2. Click Browse Integrations.
  3. In the Use an Integration dialog, select Get Subscriptions and click Create.

  4. On the Integrations page, click Create, and then click Use an Integration.

  5. On the Use an Integration dialog, select Get Subscription Products by Subscription, and then click Create.

Both integrations are now available.

Call the Integrations

Now you’ll configure the new controls to each call an integration and return values. The second control will depend on the first.

Configure the Subscription List

  1. Select the WebForm tab to display the form editor.
  2. On the form canvas, select the Subscription control.
  3. In the Properties pane, scroll down until you see the Options Source section, and select Connector.
  4. From the Connector list, select GetSubscriptions.
  5. From the Resource list, select resources.
  6. From the Operation list, select getResource.
  7. In the Response section, in the Options List field, enter response.items.
  8. In the Label Binding field, enter subscription and then select subscriptionTitle.

  9. In the Value Binding field, enter subscription and then select subscriptionNumber.

  10. Click Preview.
  11. Click the Subscription list, and notice subscription options returned from the integration appear.
  12. Click to close the preview.

Configure the Product List

  1. On the Form canvas, select the Product control.
  2. In the Properties pane, scroll down until you see the Options Source section, and select Connector.
  3. From the Connector list, select GetSubscriptionProductsBySubscription.
  4. From the Resource list, select resources.
  5. From the Operation list, select getResource.
  6. In the Template Parameters section, from the SubscriptionPUID field, select Control value.
  7. From the field next to SubscriptionPUID, select SubscriptionPUID.
  8. In the Response section, in the Options List field, enter response.items.
  9. In the Label Binding field, enter product and then select productName.
  10. In the Value Binding field, enter subscription and then select subscriptionProductPuid.
  11. Click Preview.

  12. Click the Product list.

    A message displays an error occurred retrieving the data.

  13. Click to close the preview.
  14. On the form canvas, select the Product control.
  15. In the Properties pane, in the Default Value section, select Skip Upon Load.
  16. Scroll up until you find the section before the Options Source section, and then select Hide.

    Notice that the Product control is disabled to show it's hidden.

  17. On the Form canvas, select the Subscription control.

  18. In the Properties pane, scroll down until you find the Events section.

  19. Click .

    The On Change dialog appears.

  20. From the list next to the Event Name field, select On Change.

  21. Click to display the Expression Editor dialog.

  22. Add an action to load the Product list values when the user selects a subscription:

    1. Click + Action.
    2. From the Control Name list, select ProductPUID.
    3. From the Action list, select Refresh Connector.

  23. Add an action to show the values for the Product list values when the user selects a subscription:

    1. Click + Action.
    2. From the Control Name list, select ProductPUID.
    3. From the Action list, select Show.
  24. Click OK.

  25. Let's preview the behavior you just added to the form controls:

    1. On the Form canvas, click Preview.

    2. On the Preview page, from the Subscription list, select Computer Service and Rentals -26009.

      Notice how the Product list displays its recently loaded values after the Subscription field changes.

    3. From the Product list, select GE Top Control Dishwasher.
    4. Click to close the preview.

Configure the Bill Lines Table

Let’s configure the table to display values returned from an integration, and to refresh when the selected product changes.

  1. From the Navigation menu, select Integrations.
  2. On the Application Home tab, click Create.
  3. Select Use an Integration.
  4. In the Use an Integration dialog, select Get Subscription Product Bill Lines by Product.
  5. Click Create.
  6. Select the WebForm tab to return to the Form Editor page.
  7. Select the Bill Lines table control.

  8. Configure the connector:

    1. In the Properties pane, select Use Data from Connector.
    2. From the Connector list, select GetSubscriptionProductBillLinesByProduct.
    3. From the Resource, select resources.
    4. From the Operation list, select getResource.
    5. In the Template Parameters section, from the SubscriptionPUID field, select Control value.
    6. From the field next to SubscriptionPUID, select ProductPUID.
    7. From the SubscriptionPUID field, select Control value.
    8. From the field next to SubscriptionPUID, select SubscriptionPUID.
    9. In the Response section, in the Options List field, enter response.items.
    10. Click the BillLinePuid field, and select billLinePuid.
    11. In the InvoiceText field, enter invoice and then select invoiceText.
    12. In the UsageQuantity field, enter usage and then select usageQuantity.
    13. Select Skip Upon Load.
    14. Scroll up and select Hide.
  9. On the Form canvas, select the Product control.
  10. In the Properties pane, scroll down until you find the Events section.
  11. Click .
  12. From the list next to the Event Name field, select On Change
  13. Click to display the Expression Editor dialog.

  14. Add an action that refreshes the table when the selected subscription and product value change:

    1. Click + Action.
    2. From the Control Name list, select BillLines.
    3. Leave the default selection (Self) for the Which? field as is.
    4. From the Action list, select Refresh Connector.

  15. Add an action that shows the table when the selected subscription and product value change:

    1. Click + Action.
    2. From the Control Name list, select BillLines.
    3. Leave the default selection (Self) for the Which? field as is.
    4. From the Action list, select Show.
  16. Click OK.

  17. Let's preview the behavior you just added to the table control:

    1. On the Form canvas, click Preview.

    2. On the Preview page, from the Subscription list, select Computer Service and Rentals -26009.

      Notice how the Product list displays its recently loaded values after the Subscription field changes.

    3. From the Product list, select GE Top Control Dishwasher.
    4. In the Bill Lines table, update the Usage Quantity for the first row to 100.
    5. Click to close the preview.

Automate the Update Task

Your last task is to configure an integration that updates the source system, Oracle Engagement Cloud, with all entries users make in the form.

Link an Integration

  1. From the Navigation menu, select Integrations.
  2. On the Application Home tab, click Create.
  3. Select Use an Integration.
  4. In the Use an Integration dialog, in the Search field, enter update, and then select Update Subscription Product Bill Lines.
  5. Click Create.
  6. Select the Process tab to display the process editor.
  7. Select the Update Subscription human task from the flow.
  8. Click .
  9. Insert the integration you previously linked onto the flow line.
    1. Expand the Integrations category in the BPMN palette.
    2. Drag and drop the UpdateSubscriptionProductBillLines integration to the same position on the flow as the deleted human task.

Configure the Integration’s Input

  1. Select the integration.
  2. Click .

  3. Select Open Data Association.

    The Data Association page appears.

  4. Click .

    This action automatically maps the subscription and product values users select in the form to the corresponding source fields in Oracle Engagement Cloud:

    • Process.Data Object.webFormDataObject.subscription ⟶ subscriptionPUID
    • Process.Data Object.webFormDataObject.product ⟶ subscriptionProductPUID
  5. Add a third data association for the bill lines table:
    1. In the Data Objects tree, expand Process.
    2. Expand Data Object.
    3. Expand webFormDataObject.
    4. Drag and drop billLines to the Process section.
    5. Under Update subscription product bill lines, open body.
    6. In the Update subscription product bill lines tree, expand body.

    7. Drag and drop billLine from the Update subscription product bill lines tree to the blank field that maps the value for webFormDataObject.billLines.

      The data association icon turns red and an error is displayed indicating that the arrays cannot be mapped as is.

  6. Create a transformation to map bill lines values:

    1. Click .

    2. Select Transform.

      The Transformation dialog appears.

    3. In the Transformation dialog, enter Bill Line

    4. Click Create.

      The Data Association page changes to the Transform page.

    5. Drag and drop to associate billLinePuid and billLinePUID.

    6. Drag and drop to associate string(usageQuantity and usageQuantity.

    7. Click Save.

      You exit the Transform page and return to the Data Association page.

Configure the Integration’s Output

  1. Select the Output tab.

  2. In the Data Objects tree, expand Data Object.
  3. In the Data Objects tree, click .

  4. In the Add Data Object dialog, complete these entries:

    • In the Namefield, enter updateBillLinesResponse.
    • In the Data Type section, select Business.
    • Select OracleIntegrations.UpdateSubscriptionProductBillLines.Response .
  5. Click Add.

  6. Click .

    An association between bodyOutput and updateBillLinesResponse was automatically configured.

  7. Click Apply.

    All data associations are saved, and you return to the Process tab.

Test the Process Application

Let’s see the subscription update process in action, as a user would.

  1. Click Test.

  2. Click Activate.

  3. In the Activate to Test dialog, ensure that Add Me to All Roles is selected, and click Activate.

    A confirmation message informing you that the application was successfully activated appears.

  4. Click OK.

  5. Click Try in Test Mode.

    The My Apps page appears in a new tab in your web browser.

  6. In the Search field, enter User Submit Subscription Update, and select your process .

    The form you edited displays as the first step in the process.

  7. From the Subscription list, select Computer Service and Rentals -26009.

  8. From the Product list, select GE Top Control Dishwasher.
  9. In the Bill Lines table, update the Usage Quantity for the first row to 100, and the next row to 200.

  10. Click Submit.

    An instance was created.

  11. From the Navigation menu, select My Tasks.

  12. Select the task you just created.

    Notice the quantities you just added in the form.

  13. Click APPROVE.

    The quantities are updated in the source system.
