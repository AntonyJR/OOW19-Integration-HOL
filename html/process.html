Background

In this hands-on lab, you'll create process to automate the updating of an invoice in Oracle Engagement Cloud.

We'll base this lab on the subscription billing demo discussed during other Oracle OpenWorld sessions, and on the previous lab where you created an integration.

The objective is to automate the update process to bill subscriptions for use of assets. The process you'll create will let you update the number of subscriptions for a particular product.

Create a Process

You’ll start with a predefined process and make changes to it.

  1. From the Home page’s Navigation menu, select Processes, and then select Process Applications.
  2. Click Create.
  3. In the Start with a QuickStart box, click Browse.

  4. Locate the Subscription Management QuickStart, and click its Create button.

  5. In the Create Application dialog, in the name field, enter My Subscriptions.
  6. Click Create.

  7. On the QuickStart App page, click the Switch to Application View link.

Edit the Form

You’ll add two controls to the form that each call the source system and return values for customers to choose from.

  1. On the Processes page, click the process to open it in the structured process editor.

  2. Click the purple Start activity titled Submit Subscription Update.
  3. Click and select Open Properties.

  4. In the How to identify the task? section, in the Title field, enter User Submit Subscription Update.

    Remember this name for later, when you run the process application.

  5. Close the Properties pane and select the same activity again.

  6. Click and select Open Form.

  7. Add a Select control to select the subscription:

    1. In the form editor, drag and drop a Select control from the Basic Palette to the top of the form.
    2. Select the control you just added.
    3. In the Properties pane, locate the General tab. [Carol: I know this is forced, but it was this or In the Properties pane, in the General tab, in the Name field, enter... and that seemed too long...]
    4. in the Name field, enter SubscriptionPUID.
    5. In the Label field, enter Subscription.

  8. Add a Select control to select the product:

    1. In the form editor, drag and drop a Select control below the one you just created.
    2. Select the control you just added.
    3. In the Properties pane, locate the General tab.
    4. in the Name field, enter ProductPUID.
    5. In the Label field, enter Product.
  9. Click the Preview button to see the new fields in your form.
  10. Click to close the preview.

Link the Integrations

Here you’ll add two active integrations for use in your process.

  1. From the Navigation menu, select Integrations.
  2. Click Browse Integrations.
  3. In the Use an Integration dialog, select Get Subscriptions and click Create.

  4. On the Integrations page, click Create, and then click Use an Integration.

  5. On the Use an Integration dialog, select Get Subscription Products by Subscription, and then click Create.

Both integrations are now available.

Call the Integrations

Now you’ll configure the new controls to each call an integration and return values. The second control will depend on the first.

Configure the Subscription List

  1. Select the WebForm tab to display the form editor.
  2. On the form canvas, select the Subscription control.
  3. In the Properties pane, scroll down until you see the Options Source section, and select Connector.
  4. From the Connector list, select GetSubscriptions.
  5. From the Resource list, select resources.
  6. From the Operation list, select getResource.
  7. In the Response section, in the Options List field, enter response.items.
  8. In the Label Binding field, enter subscription and then select subscriptionTitle.

  9. In the Value Binding field, enter subscription and then select subscriptionNumber.

  10. Click Preview.
  11. Click the Subscription list, and notice subscription options returned from the integration appear.
  12. Click to close the preview.

Automate the Update Task

Test the Process Application

Let’s see the subscription update process in action, as a user would.

  1. Click Test.

  2. Click Activate.

  3. In the Activate to Test dialog, ensure that Add Me to All Roles is selected, and click Activate.

    A confirmation message informing you that the application was successfully activated appears.

  4. Click OK.

  5. Click Try in Test Mode.

    The My Apps page appears in a new tab in your web browser.

  6. In the Search field, enter User Submit Subscription Update, and select your process .

    The form you edited displays as the first step in the process.

  7. From the Subscription list, select Computer Service and Rentals -26009.

  8. From the Product list, select GE Top Control Dishwasher.
  9. In the Bill Lines table, update the Usage Quantity for the first row to 100, and the next row to 200.

  10. Click Submit.

    An instance was created.

  11. From the Navigation menu, select My Tasks.

  12. Select the task you just created.

    Notice the quantities you just added in the form.

  13. Click APPROVE.

    The quantities are updated in the source system.
