Background

In this hands-on lab, you'll create an integration to update an invoice in Oracle Engagement Cloud.

We will base this lab on the subscription billing demo that we've discussed during other Oracle OpenWorld sessions.

The objective is to automate the update process to bill subscriptions for use of assets. The integration you'll create will update the number of subscriptions for a particular product.

Then, in a following lab, we'll use this integration to create an automated business process.

  1. Enter your user name and password.

    • Your user name is OICTraining1ic

Create an Integration

  1. In the Navigation menu, click Integrations, and then click Integrations.
  2. Click Create.
  3. Select App Driven Orchestration.
  4. Click Select.
  5. Name your integration: Update Subscription Product Bill Lines.
  6. (Optional) Enter a package number. For example: com.oow.UXX.1.1
  7. Click Create.
    The Integration canvas appears.

Create a Trigger

  1. Hover over START and click the plus sign (+).
  2. Click Create.
  3. Type REST in the Search textfield to find your rest connection.
  4. Select the Rest Interface connection.
    The Configure REST Endpoint dialog appears.
  5. Name your connection UpdateSubscription.
  6. Click Next.
  7. In the Resource Configuration page configure the following values:
    1. In the operation name field, enter AddLine.
    2. In the endpoint relative resource URI field, enter /subscriptions/{SubscriptionPUID}/child/products/
    3. From the action to perform list, select PATCH.
    4. Select Configure a request payload for this endpoint.
    5. Select Configure the endpoint to receive a response.
  8. Click Next.
  9. In the Request Parameters page, leave the default values, and click Next.
    The path or template parameters are automatically added and appears as the default values.
  10. On the Request page, configure the following values:
    1. From the request payload format list, select JSON Sample.
    2. Select JSON Sample for the request payload format.
    3. Click the <<<inline>>> link.
    4. Enter the following JSON sample: 
      
{
	"BillLine" : [
		{
			"BillLinePUID" : "string",
			"UsageQuantity" : "0"
		}
	]
}
    5. Click OK
    6. Click Next
  11. On the Response page, configure the following values:
    1. Select JSON Sample for the response payload format.
    2. Click the <<<inline>>> link.
    3. Enter the following JSON sample: 
      
{
	"BillLine" : [
		{
			"BillLinePUID" : "string",
			"UsageQuantity" : "0"
		}
	]
}
    4. Click OK.
    5. Click Next.
  12. In the Summary page, click Done.
    The canvas with the new trigger appears.
  13. Click Save.

Iterate Over Billing Lines

We might need to update multiple billing lines in the request. The API we'll use to connect to Oracle Engagement Cloud can only update one billing line at a time. So we'll loop over the billing lines to update a single billing line at a time.

  1. On the canvas, hover over the line that connects the first two activities, and click the plus sign (+).
  2. In the Search field, enter For.
    The search results includes connections to other system (Invokes), and activities (Actions).
  3. From the Actions section, select For Each.
    The Create Action dialog appears.
  4. In the Name field, enter LoopBillLines.
  5. Drag the BillLine element from the Source tree, and drop it on the Repeating Element field.
  6. In the Current Element Name field, enter CurrentBillLine.
  7. Click Create.
    The canvas now shows a loop.
  8. To have a better view, from the canvas toolbar, from the Layout list, select Horizontal.
  9. Click Save.

Update Billing Quantity

hThe loop we just created will update the billing lines for the request in Oracle Engagement Cloud. To do this, we will connect to Oracle Engagement Cloud.

  1. Hover over the outgoing transition for the LoopBillLines activity.
  2. In the Search textfield, enter Oracle.
  3. From the Invokes section, select Oracle CRM.
    The Configure REST Endpoint dialog appears.
  4. Name the endpoint UpdateBilling.
  5. Click Next.
  6. On the Configure the Required Operation page, leave the default values and click Next.
    The resource and verb are selected by default.
    To simplify this workshop, we've only provided a single operation. Typically, you would have to choose from multiple resources and operations.
  7. On the Configure the Request Query Parameters, click Next
  8. On the Summary page, click Done
    The Integration canvas appears.
  9. Click Save.

Add Request Mapping

The data format between applications usually do not match and that is the case here. We can use the map that was put on the canvas to transform the data from the source format to the target Engagement Cloud, format. We need to map each individual field.

  1. Click Map to UpdateBilling, and then click .
    The Map to UpdateBilling page appears. Note that there is a tooltip that indicates that there are available recommendations for this data mapping.
  2. Click  Recommendations.
    The following recommended mapping appear:
    • SubscriptionPUID ⟶ SubscriptionPUID
    • SubscriptionProductPUID ⟶ SubscriptionProductPUID
    • BillLinePUID ⟶ BillLinePUID
    • UsageQuantity ⟶ UsageQuantity
  3. Click ✓ Apply Recommendations.
